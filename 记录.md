## 项目简介

Recaping 是一款专注于多媒体录音记录的应用程序。它允许用户在录音过程中同时记录照片、视频片段和文字笔记，形成完整的多媒体时间轴记录。录音完成后，用户可以回放录音并查看所有关联的多媒体内容。

## 核心功能

1. **录音功能**：持续录音，自动分段存储（15秒/段）
2. **多媒体记录**：在录音过程中拍摄照片、录制短视频片段（5秒/段）、添加文字笔记
3. **时间轴展示**：双时间轴模型（录音时间轴 + 多媒体时间轴）
4. **回放功能**：回放录音，同步显示多媒体内容
5. **AI 功能**：录音转文字、智能摘要、关键词提取

## 开发规则与注意事项

### 代码规范

1. **命名规范**
   - 文件命名：小写 + 下划线（snake_case）
   - 类命名：大驼峰（PascalCase）
   - 变量/函数：小驼峰（camelCase）
   - 常量：全大写 + 下划线（SCREAMING_SNAKE_CASE）

2. **代码组织**
   - 按功能模块组织代码
   - 每个文件只包含一个主要类
   - 避免过长的函数，保持单一职责

3. **注释规范**
   - 公共 API 必须添加文档注释（///）
   - 复杂逻辑添加行内注释
   - TODO 标记使用 `// TODO(GDNDZZK): 描述`

### 数据库规范

1. **表设计**
   - 所有表必须包含 `id`（主键）、`created_at`、`updated_at` 字段
   - 外键使用 `_id` 后缀
   - 表名使用复数形式

2. **数据类型**
   - 时间戳使用 INTEGER（Unix 时间戳，毫秒）
   - 布尔值使用 INTEGER（0/1）
   - 大文本使用 TEXT

### 文件管理

1. **存储路径**
   - 应用数据：应用私有目录
   - 录音文件：`/audio/`
   - 照片文件：`/photos/`
   - 视频文件：`/videos/`
   - 导出文件：`/exports/`

2. **文件命名**
   - 使用时间戳 + 随机字符串
   - 格式：`{timestamp}_{random}.{ext}`

### 性能注意事项

1. **录音时**
   - 避免在主线程执行耗时操作
   - 音频写入使用异步操作
   - 定期清理内存缓存

2. **数据库操作**
   - 批量操作使用事务
   - 查询使用索引字段
   - 大数据量分页加载

3. **UI 渲染**
   - 列表使用懒加载
   - 图片使用缓存
   - 避免不必要的重建

### 安全注意事项

1. **权限管理**
   - 录音权限必须动态申请
   - 相机权限必须动态申请
   - 存储权限适配 Android 10+ Scoped Storage

2. **数据安全**
   - 敏感数据不存储在 SharedPreferences
   - 数据库文件加密（可选）
   - 导出文件注意权限控制

### 版本控制

1. **Git 提交规范**
   - 提交信息使用中文描述
   - 格式：`[类型] 描述`
   - 类型：feat, fix, docs, style, refactor, test, chore

2. **分支管理**

   - feature/*：功能分支

### 依赖管理

1. **pubspec.yaml**
